## Week 4 Progress

In the fourth week of the internship, I continued learning more features of Power BI and how they help in understanding and exploring a dataset.

One of the main things I learned this week was about Column Quality, Column Distribution, and Column Profile. These features helped me get a quick overview of the data and understand what is happening in the dataset. For example, Column Quality helped me see if there were any errors or missing values, while Column Distribution helped me understand how the values are spread in a column. Column Profile gave more detailed information about each column.

I was also introduced to the Model View in Power BI, where we can create relationships between tables. This helps when working with multiple tables because it allows the data to be connected and analyzed together.

I also learned about KPIs (Key Performance Indicators) and how they are used to measure important metrics and performance.

Another important step this week was creating my GitHub profile and organizing the project folders. I created folders for raw data, processed data, and documentation so that the project work stays organized.

I also started working on my project titled “Optimizing IT Support Team Performance Using Analytics.” For this, I began looking for datasets and understanding how the data will be prepared for analysis.

Towards the end, I was also introduced to some basic concepts of Exploratory Data Analysis (EDA), which helps in understanding a dataset by identifying patterns, trends, and basic characteristics before performing deeper analysis.

Overall, this week helped me understand more about data exploration in Power BI and how to organize and start a data analysis project.
